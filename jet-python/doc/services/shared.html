Data type for Error Handling Every API returns this under all circumstances. When there are API-specific return values, they are nested with this data type. ErrCode is 0 (EOK) when API invocation is a SUCCESS. Errcode is a negative integer when API invocation is a FAILURE. ErrStr is valid only when ErrCode != 0.
